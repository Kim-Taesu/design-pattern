# 데코레이터 패턴

#### 정의
  - 객체의 결합을 통해 기능을 동적으로 유연하게 확장 할 수 있게 해주는 패턴
  - 상속을 통해 확장 할 수 있지만, 상속은 디자인 유연성 면에서는 좋지 않다.
  
#### 단점
  1. 데코레이터를 너무 많이 사용하면 코드가 필요 잇아으로 복잡해진다.
  2. 데코레이터를 구성하는 구성요소를 초기화하는 데 필요한 코드가 훨씬 복잡해진다.
     - 구성요소의 instance만 만드는것이 안니 Wrapping 해야하는 경우도 있다. (팩토리 패턴, 빌더 패턴 사용)
  3. 데코레이터 패턴은 구성요소의 형식을 알아내서 그 결과를 바탕으로 어떤 작업을 처리하는 코드에는 적용할 수 없다.
        - 구성 요소가 데코레이터로 인해 감싸지기 때문
